В Unix (и Linux) всё есть файл, и нет различий между каталогами и файлами.
Под файлом в Linux понимается не только информация на ВЗУ, но и любое устройство.
Устройство подключается к дереву файловой системы в указанной пользователем точке с помощью команды **mount**.
Удалённый файл в Linux нельзя восстановить.

#### Типы файлов
- *Регулярные файлы* - это обычные файлы, которые содержат данные (тексты, рисунки, программный код и т.д.).
- *Канал* - это временный файл, информация из которого после чтения исчезает. Их используют для организации обмена данными между процессами.
- *Специальные файлы* соответствуют устройствам (блочные - дискам, символьные - всем прочим). Например, для чтения флэш-накопителя достаточно обычным образом открыть файл /dev/usb, позиционироваться в нем и прочесть нужное число блоков.
- *Каталоги*, позволяют объединять файлы.

#### Скрытые файлы
Скрытые файлы - это файлы, которые не отображаются при выполнении **ls**, используемые для выполнения сценариев или для хранения конфигурации служб.
Как показать скрытый файл: *ls -a*

#### Абсолютная и относительная адресация
*Абсолютная адресация* - путь к каталогу указывается, начиная с корневого каталога (**/**), и далее вниз по дереву папок до требуемого.

*Относительная адресация* - путь к каталогу указывается, начиная с текущего каталога. Когда используется относительная адресация, слэш никогда не бывает первым знаком.
